Marvis docs

Marvis is the local assistant that watches Gmail, files documents, keeps memory, and talks over Telegram.
Standalone file: docs/index.html
Open dashboard: 127.0.0.1:8080 Docs route: /docs

What Marvis does

Assistant loop

Marvis answers Telegram messages through the agent loop, with memory, Drive search, calendar, task, and financial tools available when needed.

Background watcher

It polls Gmail on a timer, decides whether a message is worth filing, extracts attachments, classifies them, and stores them in Drive plus memory.

Setup and how to run

  1. Install Python 3.12, create a virtual environment, and install the requirements.
  2. Copy .env.example to .env and fill in OpenRouter, Telegram, and Google credentials.
  3. Create Google OAuth credentials with Gmail, Drive, and Calendar scopes, then authorize once so token.json is created.
  4. Start the app with python main.py.
  5. Open the dashboard at http://127.0.0.1:8080/ and this docs page at /docs.

Telegram usage

User messageplain text goes into the agent loop
Commands/status, /memories, /forget, /reset
Uploadsdocuments and photos are filed to Drive
Any textPassed to the agent as a normal chat message.
/memoriesLists stored memories grouped by category.
/forget <topic>Deletes a memory by topic.
/resetClears conversation history only. Long-term memory stays intact.
/statusShows memory count, Drive state, Calendar state, and current model.
Telegram is single-user by design: the bot only accepts messages from the configured allowed user ID.

Gmail watcher behavior

1. Pollreads unread mail every GMAIL_POLL_INTERVAL seconds
→
2. Parseextracts text, metadata, and attachments
→
3. Relevanceskips newsletters, OTPs, and noisy notifications
→
4. Fileclassify attachment, upload to Drive, write memory
Statedata/gmail_state.txt keeps the last processed message id and cutoff date.
Activitydata/gmail_activity.jsonl captures filed, skipped, partial, and failed outcomes.
Logslogs/jarvis.log records watcher activity and filing decisions.
If the relevance or filing step fails, Marvis should lean toward not losing a document. The watcher is built around preserving useful mail, not silently discarding it.

Memory, Drive, calendar, and task architecture

MemorySQLite memories table + ChromaDB semantic index
TasksSQLite tasks table, created and completed through tools
Financial recordsSQLite financial_records table linked to filed Drive docs

Memory

Structured facts, preferences, decisions, document references, projects, household items, finances, health notes, and tasks live in SQLite, while ChromaDB provides semantic recall.

Drive

Files land under the configured Google Drive root, with top-level folders and model-driven subfolders. The agent can also search and read Drive files directly.

Calendar

Calendar reads and writes use Google Calendar via the calendar client. Relative dates like today or tomorrow are resolved before calling the API.

Tasks

Tasks stay in SQLite, with explicit create/list/complete tool calls. They are only created when the user asks for a task or reminder.
